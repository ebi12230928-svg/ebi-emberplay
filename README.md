# EMBERPLAY

エンタメ専用ポイント(Embers)を使った、Stake.com / Shuffle.com 風のミニゲームサイトです。

**重要: Embersは無料の報酬でのみ入手できます。現金・ギフトカード・暗号資産・景品など、価値のあるものへの交換機能は一切なく、今後も追加しない前提で設計されています。この前提を変えると賭博に該当する可能性があるため、換金・出金機能は絶対に追加しないでください。**

## セットアップ(ローカル)

```bash
pip install -r requirements.txt
python app.py
```

起動後、`http://localhost:5000` にアクセスしてください。初回起動時に `emberplay.db`(SQLite)が自動生成されます。

## 最初の管理者を作る

1. サイトで通常どおり新規登録する
2. `app.py` の中の以下の行を、自分のユーザー名に書き換える

```python
initial_admin_username = os.environ.get("INITIAL_ADMIN_USERNAME", "ここに自分のユーザー名")
```

3. サーバーを再起動する(コンソール操作は不要です。起動するたびに、このユーザーがまだ管理者でなければ自動的に管理者になります)

管理者になると `/admin` から他のユーザーへのポイント付与・借金の設定・コードの発行・管理者権限の付与ができます。

## 実装済みの機能

- 会員登録・ログイン(独自アカウント、Discordとは連携していません)
- ウォレット: デイリー / ウィークリー / マンスリー / リロード / レーキバック報酬の受け取り、コード引き換え(`/redeem`)
- プレイ履歴(`/history`)・ランキング(`/leaderboard`、残高top3・最大獲得top3)
- お知らせ通知(ベルアイコン、`/notifications`)
- 管理者パネル(`/admin`):
  - 個別へのポイント付与/回収
  - **全プレイヤーへの一斉ポイント配布/回収**(本人に通知が届きます)
  - **借金の設定/免除**(下記「借金機能について」を参照)
  - **コードの発行**(「1人1回まで」または「全体で合計N回まで」の2種類、発行時に全員へ通知)
  - 管理者権限の付与/解除
- プロヴァブリーフェア: サーバーシード(事前ハッシュ公開)・クライアントシード・nonceによる検証可能な乱数
- ゲーム(全35種): Dice, Limbo, Crash, Mines, Plinko, Keno, Wheel, HiLo(パス機能付き), Dragon Tower, Coin Flip, Sic Bo, War, Roulette, American Roulette(00あり), Blackjack, Baccarat, Video Poker(Jacks or Better), Red Dog, Andar Bahar, Craps(Pass/Don't Pass), Three Card Poker、そしてスロット14テーマ(Cherry Classic, Egyptian Gold, Pirate's Treasure, Space Odyssey, Ocean Deep, Wild West, Neon Nights, Jungle Fever, Candy Pop, Norse Mythology, Lucky Sevens, Dragon's Hoard, Retro Arcade, Diamond Vault)
  - 前回より全体的に難易度(house edge)を引き上げています
- **スポーツベット**(`/sportsbook`): 実在のサッカーの試合(プレミアリーグ・ラ・リーガ・ブンデスリーガ・チャンピオンズリーグ)に賭けられます。詳細は下記「スポーツベット機能について」を参照
- **VIPラウンジ**(`/vip-lounge`): 管理者が手動でVIPを付与したユーザーのみアクセス可能。試合日程・結果に加え、管理者だけが投稿できるお知らせ欄があります
- **チャット**(`/chat`): 全プレイヤーが参加できる、数秒おきに自動更新されるチャット欄

## 報酬の拡充について

- **アワリー報酬**を新設(1時間ごとに受け取り可能・100 Embers)
- **ログインボーナス**を新設(連続ログイン日数に応じて増額、7日目以降は7日目の額でループ。1日休むとリセット)
- デイリー・ウィークリー・マンスリー・リロードの受け取り額をそれぞれ引き上げました(`config.py`で調整可能)

## ロビー画面のカルーセル化について

Stakeのような、横にスライドするカード形式に作り直しました。

- **プレーを続ける**: 自分の直近のプレイ履歴(重複なし・新しい順)から自動生成されます。まだ何もプレイしていない場合は表示されません
- **あなたへのおすすめゲーム**: 全35ゲームの中からランダムに8つ選ばれます。**ページを開くたびに毎回変わります**
- 各カテゴリー(オリジナル/テーブルゲーム/スロット)も横スクロール形式になり、「すべて見る」から全件表示ページに移動できます
- 検索ボックスは引き続き使えます(該当しないカードは自動的に隠れます)

## 新しいゲームについて

- **American Roulette**: ヨーロピアンルーレットに00を追加したバージョン。配当はヨーロピアン版と同じですが、ポケット数が1つ増える分、的中率がわずかに下がります
- **Red Dog**: 2枚のカードの間に3枚目が入るかを予想するシンプルなカードゲーム。全パターンを実際に総当たりして期待値を検証済みです(house edge約10.6%)
- **Andar Bahar**: ジョーカーと同じ数字のカードが「Andar」「Bahar」どちらに先に出るかを予想するインドの伝統的なカードゲーム。確率を正確に計算した上で配当を設定しています(どちらも約5.9%のhouse edge)
- **Craps**: Pass LineまたはDon't Passを選んでサイコロを振る、カジノの定番ゲームの簡易版です
- **Three Card Poker**: Ante(基本ベット)を賭けて3枚配られ、Play(続行)かFold(降参)を選ぶポーカー系ゲーム。役がStraight以上ならAnteボーナスも付きます。役の判定ロジック(Straight Flush・Three of a Kind・Straight・Flush・Pair・High Card、3枚特有のQ-K-Aやウィーラー処理も含む)は実際にテストして正しく動作することを確認済みです

## 借りる機能について

- ウォレット画面(`/wallet`)から、誰でも自分から指定額(`config.py`の`LOAN_AMOUNT`、デフォルト3,000)Embersを借り入れられます
- すでに借金がある間は、完済するまで再度借りることはできません
- それ以外の返済ルール(勝利分が返済に充当される、完済時に1%だけ残高化される、1ヶ月未返済でマンスリー凍結、など)は管理者が設定する借金と同じです

## お知らせ掲示板について

- `/announcements` ページで、管理者が投稿したお知らせを全プレイヤーが閲覧できます(VIP限定ではありません)
- 管理者パネル(`/admin`)から投稿・削除ができ、投稿すると全プレイヤーに通知が届きます
- ロビー画面の上部にも、最新3件が表示されます
- これはVIPラウンジ内のお知らせ欄(VIP限定)とは別物です。使い分けの目安:全員向けのお知らせ→掲示板、VIP向けの特別な告知→VIPラウンジ

## アニメーションについて

- **残高(オドメーター)**: 数字が変わった桁だけがパタパタとめくれるアニメーションに変更しました
- **各ゲームの勝敗表示**: 勝ったときはポップして光る演出、負けたときは横に揺れる演出を、全ゲーム共通で追加しました(`static/js/balance.js`の`EmberPlay.flashResult()`という共通関数を使っています)
- **Mines/Dragon Towerのマス開封**: 安全なマスはポップイン、地雷を踏んだマスは揺れるアニメーション
- **借金パネル**: 借金が残っている間、パネルの枠がゆっくり脈打つ演出
- **報酬受け取り成功時**: ウォレット画面のカードが軽くポップする演出

## VIPラウンジについて

- 管理者パネル(`/admin`)から、ユーザーごとに「VIPにする/解除」を切り替えられます(自動昇格の条件などは現状ありません)
- VIPになると本人に通知が届き、ナビに金色の「VIPラウンジ」リンクが表示されます
- ラウンジ内の「お知らせ」欄は管理者のみ投稿・削除できます(例: 「今夜21:00からDAZNでプレミアリーグの放送があります」といった告知に利用できます)。**外部の配信サービス(DAZNなど)の映像そのものを埋め込むことは著作権上できないため、あくまで告知・案内にとどめています**
- 試合データはスポーツベット機能と同じTheSportsDBのデータを再利用しています

## チャット機能について

- 全ログインユーザーが利用できます(VIP限定ではありません)
- 本格的なWebSocketではなく、**2.5秒ごとに新着メッセージを取得するポーリング方式**です。RailwayやRenderのような常時稼働型のホスティングだけでなく、PythonAnywhereやVercelのようなサーバーレス環境でも問題なく動作するよう、あえてこの方式にしています
- 荒らし対策として、1人あたり1.5秒に1回までの投稿制限と、1メッセージ500文字までの制限を設けています
- 管理者・VIPのメッセージには、それぞれ色分けとラベルが表示されます

## スポーツベット機能について

[TheSportsDB](https://www.thesportsdb.com)という無料のスポーツデータAPIを使い、実際の試合結果を取得しています。

- 無料の共通APIキー(`123`)を使用しているため、**登録は不要**です
- ページを開くたびに最新の試合日程・結果を取得しますが、無料枠を節約するため**5分に1回だけ**外部APIを呼び出す仕組みになっています
- 賭けたあと試合が終了すると、次に誰かが`/sportsbook`ページを開いたタイミングで自動的に結果が判定され、的中していればEmbersが付与されお知らせが届きます
- **オッズはチームの強さ(現在の順位表の勝ち点)に応じて自動計算されます**。勝ち点差が大きいほど、強い方の配当は下がり、弱い方の配当は上がります(どのオッズでも約8%のhouse edgeになるよう正規化済みです)。Champions Leagueのように順位表を参照できない大会は、暫定的にデフォルトのオッズ(勝ち1.9倍・引き分け3.2倍)になります
- 無料APIキーは共有のものなので、アクセスが集中する時間帯は取得に失敗することがあります(その場合は該当リーグの更新がスキップされるだけで、エラーにはなりません)

## ゲームごとの配当倍率調整について

- 管理者パネル(`/admin`)の一番上に、全35ゲーム分の「配当倍率スケール」の一覧があります
- 1.0が通常の状態です。0.8に設定するとそのゲームの配当が全体的に20%下がり(難しくなり)、1.2に設定すると20%上がります(甘くなります)
- 設定できる範囲は0.1x〜3.0x です
- ゲーム内部の当選確率(サイコロの出目やカードの引き方など)自体は変更されません。あくまで「当たったときにもらえる量」だけを一括調整する仕組みです
- スロットは14テーマありますが、管理を簡単にするため全テーマ共通の倍率1つで調整する仕様にしています(テーマごとに個別設定したい場合はご依頼ください)
- 引き分け・プッシュ(賭け金がそのまま返ってくる結果)は調整の対象外です(常に賭け金がそのまま返ります)

## 借金機能について(重要・仕様の確認をお願いします)

ご依頼内容の説明がやや解釈の余地があったため、以下のように実装しました。想定と違う場合は教えてください、調整します。

1. 管理者が `/admin` から特定ユーザーに借金を設定すると、**そのユーザーの現在の残高は0になり**(=すべて「借金ポイント」として没収)、借金額が記録されます。本人に通知が届きます。
2. 借金がある間、**ゲームでの勝利分は残高に反映されず、全額が借金の返済に充てられます**。デイリー/ウィークリーなどの無料報酬は今までどおり通常に受け取れます(遊ぶための元手として使えます)。
3. ちょうど借金を完済した回に勝利分が余った場合、その余った分の**1%だけ**が残高として反映され、以降は通常どおりプレイできます。
4. **借金を1ヶ月以内に返済しなかった場合、マンスリー報酬が借金完済まで使えなくなります**。
5. 管理者は「借金を免除する」でいつでも借金をリセットできます。

## コード機能について

- **「1人1回まで」タイプ**: 何人でも使えますが、同じ人は1回しか使えません。発行時に全員へお知らせが届きます。
- **「全体で合計N回まで」タイプ**: 1人1回という制限はなく、合計でN回使われたら終了です(同じ人が複数回使うことも可能です)。こちらも発行時に全員へお知らせが届きます。

## 学校などフィルターの効きやすい環境での利用について

ブランド名を「EMBERPLAY」、通貨を「Embers」とし、UI上の文言も「賭け金」→「プレイ料」、「ベット」→「選択」のように、賭博を直接連想させる単語を避けるよう調整しています。

ただし、これは**完全な保証ではありません**。学校などのネットワークフィルター(Bark、GoGuardian、Securlyなど)は、単語だけでなくドメインの評判・カテゴリ分類・画像解析など複数の方法でサイトを判定するため、必ず通過できるとは限らない点をご理解ください。心配な場合は、実際に導入予定のネットワークで事前にアクセステストすることをおすすめします。

## Renderへのデプロイ手順(クレジットカード登録不要)

[Render](https://render.com) は、Railwayと似た使い勝手(GitHub連携→自動デプロイ)でありながら、**無料プランの利用にクレジットカード登録が不要**なサービスです(2026年時点で複数の情報源で確認済み)。

**知っておいてほしい制限**
- 無料プランは15分アクセスがないとスリープし、次のアクセス時に30〜60秒ほど起動待ちが発生します
- Render自体の無料Postgresは期限付き(30〜90日で削除)なので、**Vercelの手順で案内したNeon/SupabaseのPostgresをそのまま使う**ことを強くおすすめします(同じ`DATABASE_URL`をそのまま使い回せます)

**手順**

1. **外部のPostgresを用意する**(Vercel用に作成済みなら、そのまま使えます。まだなら[Neon](https://neon.tech)などで無料作成し、接続URLを控えておく)
2. **GitHubリポジトリを作る**(すでにお持ちなら、このままでOKです)
3. **Renderでアカウント作成**: [render.com](https://render.com) にGitHub/Google等でサインアップ(クレジットカード不要)
4. **New +** →「Web Service」→ 対象のGitHubリポジトリを選択
5. このプロジェクトには `render.yaml` を同梱しているので、Renderが自動的に設定を読み込みます(読み込まれない場合は、以下を手動で入力してください)
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `gunicorn wsgi:app`
6. **環境変数を設定**(「Environment」タブ):
   - `DATABASE_URL` = 手順1で控えたPostgresの接続URL
   - `SECRET_KEY` = ランダムな文字列(`render.yaml`により自動生成される場合はそのままでOK)
   - `INITIAL_ADMIN_USERNAME` = 最初に管理者にしたいユーザー名
7. **Create Web Service** を押してデプロイ完了を待つ
8. 発行される `xxxx.onrender.com` のURLでアクセスできます

## Vercelへのデプロイ手順(ドメイン取得なしでOK)

**重要な前提**: Vercelは「サーバーレス」という仕組みで動くため、SQLiteのようなファイルベースのデータベースは使えません(リクエストのたびにデータが消えます)。**必ず外部のPostgresデータベースを用意してください**(下記手順内で案内します)。

1. **外部のPostgresデータベースを用意する**: [Neon](https://neon.tech)や[Supabase](https://supabase.com)は無料枠があり、数分でPostgresデータベースを作成できます。作成後、接続用のURL(`postgresql://...`から始まる文字列)を控えておいてください。
2. **GitHubリポジトリを作る**: このプロジェクトのファイル一式をGitHubリポジトリにpushしてください。
3. **Vercelでプロジェクト作成**: [vercel.com](https://vercel.com) にログイン(GitHubアカウントでのログインが簡単です)→「Add New...」→「Project」→ 先ほどのリポジトリをImport。
4. **環境変数を設定**: プロジェクトの「Settings」→「Environment Variables」で以下を追加してください。
   - `DATABASE_URL` = 手順1で控えたPostgresの接続URL
   - `SECRET_KEY` = ランダムな長い文字列(推測されないもの)
   - `INITIAL_ADMIN_USERNAME` = 最初に管理者にしたいユーザー名
5. **デプロイ**: 「Deploy」を押すだけです。Vercelは `requirements.txt` からPythonアプリだと自動検出し、`pyproject.toml` に書かれた `wsgi:app` をエントリポイントとして起動します。静的ファイル(CSS/JS)は `public/static/` に同梱済みで、Vercelが自動的にCDN経由で配信します。
6. **公開URLの確認**: デプロイが完了すると `xxxx.vercel.app` の形式のURLが発行されます。これが公開URLです。
7. **(任意)独自ドメインを後から追加したい場合**: プロジェクトの「Settings」→「Domains」からお持ちのドメインを紐づけられます。

### コードを更新したいとき
GitHubにpushするだけで、Vercelが自動的に再デプロイしてくれます。

### Vercel特有の制限について
- 1回のアクセスあたりの処理時間に上限があります(`vercel.json`で30秒に設定済み)。通常の利用では問題になりません。
- アプリ全体のサイズが500MBを超えないようにしてください(現状のEMBERPLAYは十分小さいです)。

## (代替案)Railwayへのデプロイ手順

サーバーレスではなく、常時起動する通常のサーバーとして動かしたい場合はRailwayも使えます。こちらはSQLiteでもある程度動きますが、やはりPostgresの利用を推奨します。

1. **GitHubリポジトリを作る**: プロジェクト一式をGitHubにpush。
2. **Railwayでプロジェクト作成**: [railway.app](https://railway.app) →「New Project」→「Deploy from GitHub repo」→ リポジトリを選択。
3. **データベースの追加(推奨)**: 「+ New」→「Database」→「PostgreSQL」を追加。自動的に `DATABASE_URL` が渡されます。
4. **環境変数を設定**: 「Variables」タブで `SECRET_KEY` と `INITIAL_ADMIN_USERNAME` を追加。
5. **デプロイ**: `Procfile`(`web: gunicorn wsgi:app --bind 0.0.0.0:$PORT`)を自動検出して起動します。
6. **公開URLの確認**: 「Settings」→「Networking」から `xxxx.up.railway.app` のURLが発行されます。

## 設定値の変更

`config.py` の `Config` クラスで、報酬額・クールダウン時間・レーキバック率などを調整できます。各ゲームの house edge(控除率)はそれぞれの `games/*.py` 内で個別に設定しています。

## 本番運用時の注意

- `config.py` の `SECRET_KEY` を推測されない値に変更してください(環境変数 `SECRET_KEY` で上書き可能です)。
- Railwayなど本番環境ではPostgreSQLの利用を推奨します(上記デプロイ手順を参照)。
- HTTPS環境での運用を推奨します(Railwayは標準でHTTPSが有効になります)。
